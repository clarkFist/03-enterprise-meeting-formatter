## 🎯 工具简介

### 这个工具是做什么的？

这是一个专门为**CASCO SIGNAL VCU项目**设计的会议纪要自动生成工具。它能帮你：

1. **📸 图片识别参会人员**：拍照或截图参会人员统计表，自动识别姓名、工号、角色
2. **📝 智能解析会议内容**：输入或粘贴会议记录文本，自动提取关键信息
3. **🎨 自动生成专业纪要**：生成规范的Markdown格式会议纪要
4. **📕 一键导出PDF**：转换为企业级排版的PDF文档，可直接打印分发

### 核心优势

| 特点 | 传统方式 | 使用本工具 |
|------|---------|-----------|
| **参会人员录入** | 手动逐个输入 | 拍照自动识别（秒级） |
| **会议内容整理** | 手动分类整理 | 智能结构化解析 |
| **格式规范** | 手工排版 | 自动标准化格式 |
| **生成时间** | 1-2小时 | 1-2分钟 |
| **准确性** | 依赖人工 | AI辅助+数据库匹配 |

### 技术特性

- ✨ **AI驱动**：使用Claude Vision API和Claude API智能识别和解析
- 🎯 **数据库匹配**：内置16人VCU项目参会人员数据库，自动匹配补全信息
- 🔄 **双输入模式**：分离处理图片和文本，更高准确度
- 📋 **模板化输出**：基于Jinja2模板，确保格式统一
- 🎨 **企业级PDF**：使用Chromium引擎渲染，专业排版

## 🚀 快速开始

### 第一步：准备环境

#### 1.1 检查Python版本

```bash
python3 --version
# 需要 Python 3.7 或更高版本
```

#### 1.2 设置API密钥

```bash
# 设置Claude API密钥（必需）
export ANTHROPIC_API_KEY='your-api-key-here'

# 永久保存到配置文件（推荐）
echo 'export ANTHROPIC_API_KEY="your-api-key-here"' >> ~/.zshrc
source ~/.zshrc
```

> 💡 **获取API密钥**：访问 https://console.anthropic.com/ 注册并获取API密钥

本工具的Shell包装脚本会自动检查并安装依赖，无需手动操作。

### 第二步：准备输入文件

#### 2.1 准备参会人员图片

**支持的格式**：PNG、JPG、JPEG、GIF、WebP

**图片示例说明**：

**拍照/截图要求**：
- ✅ 清晰可读（不模糊、不倾斜）
- ✅ 包含姓名、工号（必需）

### 第三步：一键生成

**最简单的方式**：

```bash
cd skill-package/markdown-pdf-converter

./scripts/ocr-wrapper.sh 参会人员.png \
    --content meeting-notes.txt \
    --generate-pdf
```

**就这么简单！** 🎉

执行后会自动：
1. 识别图片中的参会人员（3-5秒）
2. 解析文本中的会议内容（5-10秒）
3. 匹配VCU项目数据库（1秒）
4. 生成YAML配置文件
5. 生成Markdown会议纪要
6. 转换为企业级PDF文档
7. 自动打开PDF供查看

**完成时间**：通常15-25秒

## 📚 详细使用指南

### 使用方式一：Shell包装脚本（推荐）

Shell脚本提供最简单的使用体验，自动处理环境检查和依赖安装。

#### 基本语法

```bash
./scripts/ocr-wrapper.sh <图片路径> [选项]
```

#### 选项说明

| 选项 | 说明 | 示例 |
|------|------|------|
| `-c, --content FILE` | 会议内容文本文件路径 | `--content notes.txt` |
| `-t, --text TEXT` | 直接输入会议内容文本 | `--text "会议时间..."` |
| `-o, --output FILE` | 指定输出配置文件路径 | `-o config.yaml` |
| `--generate-pdf` | 自动生成PDF（推荐） | `--generate-pdf` |
| `--no-open` | 不自动打开生成的PDF | `--no-open` |
| `-h, --help` | 显示帮助信息 | `--help` |

#### 使用示例

**示例1：标准用法（图片+文本文件）**

```bash
./scripts/ocr-wrapper.sh attendees.png \
    --content meeting-notes.txt \
    --generate-pdf
```

**示例2：直接输入文本**

```bash
./scripts/ocr-wrapper.sh attendees.png \
    --text "会议时间: 2025-11-14 14:00-16:00
会议地点: 企业微信会议
一、项目整体情况..." \
    --generate-pdf
```

**示例3：仅生成配置文件（不生成PDF）**

```bash
./scripts/ocr-wrapper.sh attendees.png \
    --content meeting-notes.txt \
    -o custom-config.yaml
```

生成配置文件后，可手动编辑，然后再生成纪要：

```bash
# 编辑配置
vim custom-config.yaml

# 生成纪要
python scripts/generate-meeting.py custom-config.yaml

# 转换PDF
./scripts/convert.sh RB99125046*.md enterprise
```

**示例4：批量生成（不自动打开）**

```bash
./scripts/ocr-wrapper.sh meeting1.png \
    --content notes1.txt \
    --generate-pdf --no-open

./scripts/ocr-wrapper.sh meeting2.png \
    --content notes2.txt \
    --generate-pdf --no-open
```

### 使用方式二：Python直接调用

适合需要更多控制或集成到其他脚本的场景。

#### 基本语法

```bash
python3 scripts/ocr-meeting-extractor.py <图片路径> [选项]
```

#### 选项与Shell脚本相同

```bash
python3 scripts/ocr-meeting-extractor.py attendees.png \
    --content meeting-notes.txt \
    --generate-pdf
```

### 使用方式三：分步操作（高级）

适合需要手动调整配置的场景。

#### 步骤1：提取信息生成配置

```bash
./scripts/ocr-wrapper.sh attendees.png \
    --content meeting-notes.txt

# 输出：data/meeting-input-20251114_150230.yaml
```

#### 步骤2：编辑配置文件（可选）

```bash

vim data/meeting-input-20251114_150230.yaml

# 或
code data/meeting-input-20251114_150230.yaml
```

可调整内容：
- 修正参会人员信息
- 调整会议时间地点
- 补充或修改模块进展
- 添加任务和风险

#### 步骤3：生成Markdown纪要

```bash
python scripts/generate-meeting.py data/meeting-input-20251114_150230.yaml

# 输出：RB99125046安全运算与控制平台（VCU）项目例会会议纪要_20251114.md
```

#### 步骤4：转换为PDF

```bash

# 输出：RB99125046安全运算与控制平台（VCU）项目例会会议纪要_20251114.pdf
```

## 🎬 使用场景

### 场景1：常规周会纪要生成

**背景**：每周三下午VCU项目例会，需要生成标准会议纪要。

**操作流程**：

1. **会议前**：无需准备（数据库已有人员信息）

2. **会议中**：
   - 截图或拍照参会人员统计表
   - 记录会议内容到notes.txt

3. **会议后**（1分钟完成）：
   ```bash
   ./scripts/ocr-wrapper.sh 20251114_参会人员.png \
       --content 20251114_会议记录.txt \
       --generate-pdf
   ```

4. **分发**：PDF自动打开，直接分发给团队

**时间节省**：从1小时 → 1分钟

### 场景2：紧急会议快速记录

**背景**：突发技术问题临时会议，需要快速生成纪要存档。

**操作流程**：

```bash
# 直接使用命令行输入，无需创建文件
./scripts/ocr-wrapper.sh 临时会议人员.png \
    --text "紧急技术会议
时间：2025-11-14 16:30
问题：生产环境CPU占用率异常
决策：立即回滚到v1.2版本
负责人：刘浩洋
截止：今晚20:00" \
    --generate-pdf
```

**优势**：无需创建文件，直接生成纪要

### 场景3：历史会议补录

**背景**：发现上周会议纪要未生成，需要补录。

**操作流程**：

```bash
# 使用历史照片和记录
./scripts/ocr-wrapper.sh 上周会议照片.png \
    --content 上周记录.txt \
    -o data/meeting-20251107.yaml

# 手动调整日期和内容
vim data/meeting-20251107.yaml

# 生成纪要
python scripts/generate-meeting.py data/meeting-20251107.yaml
./scripts/convert.sh RB99125046*.md enterprise
```

### 场景4：多场次会议批量处理

**背景**：月末整理，需要生成本月所有会议纪要。

**批量脚本**：

```bash
#!/bin/bash
# batch-generate.sh

for i in {1..4}; do
    week="week$i"
    ./scripts/ocr-wrapper.sh "photos/${week}_attendees.png" \
        --content "notes/${week}_content.txt" \
        --generate-pdf --no-open
    echo "✅ 第${i}周会议纪要已生成"
done

echo "🎉 所有会议纪要生成完成"
```

## ❓ 常见问题

### Q1: API密钥错误怎么办？

**错误信息**：
```
❌ 未设置 ANTHROPIC_API_KEY 环境变量
```

**解决方案**：
```bash
# 方式1：临时设置（当前终端有效）
export ANTHROPIC_API_KEY='your-api-key'

# 方式2：永久设置（推荐）
echo 'export ANTHROPIC_API_KEY="your-api-key"' >> ~/.zshrc
source ~/.zshrc

# 验证设置
echo $ANTHROPIC_API_KEY
```

### Q2: 依赖安装失败？

**错误信息**：
```
❌ 安装 anthropic 失败
```

**解决方案**：
```bash
# 方式1：手动安装
pip3 install anthropic PyYAML Jinja2 Pillow

# 方式2：使用国内镜像
pip3 install -i https://pypi.tuna.tsinghua.edu.cn/simple anthropic PyYAML Jinja2 Pillow

# 方式3：升级pip
pip3 install --upgrade pip
pip3 install anthropic PyYAML Jinja2 Pillow
```

### Q3: 图片识别不准确？

**可能原因和解决方案**：

| 问题 | 解决方案 |
|------|---------|
| 图片模糊 | 使用清晰的截图或照片 |
| 图片倾斜 | 保持图片水平拍摄 |
| 文字太小 | 放大后再截图 |
| 光线不足 | 使用屏幕截图代替拍照 |
| 格式不规范 | 使用标准表格格式 |

**最佳实践**：
- ✅ 使用屏幕截图（最清晰）
- ✅ 确保文字完整显示
- ✅ 避免手写内容
- ✅ 使用标准字体

**提示信息**：

### Q6: PDF生成失败？

**错误信息**：
```
⚠️  PDF生成失败
```

**检查清单**：
1. 确认Markdown文件已生成
2. 检查内置转换依赖是否安装（markdown/pyppeteer/pygments）
3. 验证Chromium下载是否完成（pyppeteer 首次会自动下载）

**解决方案**：
```bash
# 检查Markdown是否存在
ls -lh RB99125046*.md

# 手动转换PDF（内置）
./scripts/convert.sh RB99125046*.md enterprise

# 如果还失败，尝试直接用 Python 调用
python - <<'PY'
from pathlib import Path
import sys
skill_dir = Path("skill-package/markdown-pdf-converter").resolve()
sys.path.insert(0, str(skill_dir / "lib"))
from pdf_converter import convert_markdown_to_pdf
print(convert_markdown_to_pdf("RB99125046安全运算与控制平台（VCU）项目例会会议纪要_20251115.md", theme="enterprise", auto_open=False))
PY
```

### 技巧2：批量处理脚本

创建 `batch-generate.sh`：

```bash
#!/bin/bash

# 配置目录
PHOTO_DIR="meeting-photos"
NOTES_DIR="meeting-notes"
OUTPUT_DIR="outputs/$(date +%Y-%m)"

# 创建输出目录
mkdir -p "$OUTPUT_DIR"

# 遍历处理
for photo in "$PHOTO_DIR"/*.png; do
    filename=$(basename "$photo" .png)
    notes="$NOTES_DIR/${filename}.txt"

    if [ -f "$notes" ]; then
        echo "📋 处理: $filename"
        ./scripts/ocr-wrapper.sh "$photo" \
            --content "$notes" \
            --generate-pdf --no-open

        # 移动输出文件
        mv RB99125046*.md "$OUTPUT_DIR/" 2>/dev/null
        mv RB99125046*.pdf "$OUTPUT_DIR/" 2>/dev/null
        mv data/meeting-input-*.yaml "$OUTPUT_DIR/" 2>/dev/null

        echo "✅ 完成: $filename"
    else
        echo "⚠️  跳过: $filename (缺少文本文件)"
    fi
done

echo "🎉 批量处理完成，输出目录: $OUTPUT_DIR"
```

使用：
```bash
chmod +x batch-generate.sh
./batch-generate.sh
```

## 📝 最佳实践总结

### DO（推荐做法）✅

1. **使用双输入模式**：图片+文本分离，准确度更高
2. **保持图片清晰**：使用屏幕截图而非照片
3. **结构化文本**：使用明确的标识符和分类
4. **及时生成**：会议结束后立即生成，信息最新鲜
5. **检查配置**：生成PDF前快速浏览YAML配置
6. **定期备份**：保存历史会议纪要和配置文件
7. **更新数据库**：新员工加入时及时更新attendees.yaml

### DON'T（避免做法）❌

1. ❌ 不要使用模糊或倾斜的图片
2. ❌ 不要在文本中使用过于自由的格式
3. ❌ 不要忘记设置API密钥
4. ❌ 不要同时使用 `--content` 和 `--text` 参数
5. ❌ 不要在生产环境测试（先在测试图片上验证）
6. ❌ 不要跳过配置文件检查就分发PDF
7. ❌ 不要将API密钥硬编码到脚本中

### 效率提升技巧

1. **创建模板文件**：为常规会议创建文本模板，只需填充内容
2. **使用快捷命令**：设置alias简化命令输入
3. **批量处理**：用脚本一次性处理多个会议
4. **保存常用配置**：复用上次会议的参会人员列表
5. **自动化流程**：集成到会议管理系统中
